# 跨境电商 AI 图片工作台

一个在本地浏览器中运行的跨境电商商品资料整理、图片策划、AI 生图和批量项目管理工具。

当前版本在原有 Amazon Listing / A+ 图片工作流上，增加了平台通用模式、自由创作模式、商品资料助手、灵活逐图方案、参考套图重构、批量项目队列和项目 ZIP 归档。

## 1. 主要功能

### 商品资料与文案

- 允许只输入已知的材质、尺寸、用途、颜色、结构和卖点，不要求一开始就有完整标题和五点描述。
- 将零散信息整理成“已确认事实、待确认推测、缺失信息、矛盾信息”。
- AI 推测不会自动当作商品事实，需要人工勾选确认。
- 可生成 Amazon 标题、Amazon 五点描述、通用短描述、通用详细描述和通用卖点。
- 支持填写目标输出语言，适合跨境电商多语言文案。

### 三种创作模式

| 模式 | 适用场景 | 主要特点 |
| --- | --- | --- |
| Amazon 合规 | Amazon Listing、A+ Content | 保留固定图片位、A+ 模块、主图白底及平台合规限制 |
| 平台通用 | Walmart、eBay、Temu、独立站等 | 可填写目标平台和平台规则，不继承 Amazon 专属限制 |
| 自由创作 | 海报、社媒图、品牌图、带 Logo 或水印的图片 | 图片数量、文字、Logo、水印、构图和风格更自由 |

### 图片策划与生成

- 逐图设置图片用途、画面目标、构图、图片文案、比例和单张额外要求。
- 支持 1–12 张图片方案。
- 支持新增、复制、上移、下移和删除图片卡片。
- 修改方案后需要重新确认，防止误用旧方案。
- 可将选中的单张方案编译成结构化提示词并填入生图栏。
- 用户的整套要求和单张要求拥有明确优先级。

### 参考套图与批量项目

- 上传参考套图后，可按参考图数量建立一一对应的新套图方案。
- 支持轻度重构、标准重构和深度原创三个方向。
- 支持 5 个独立批量项目槽位，可单选、多选或全选。
- 同时最多处理 2 个项目，每个项目最多同时生成 2 张图片。
- 单个项目失败不会阻断其他项目。
- 刷新后可根据任务记录恢复已提交任务状态。
- 失败重试只补生成失败或缺失的图片，不重复生成已经成功的图片。

### 本地归档

- 每个批量项目可单独导出 ZIP。
- ZIP 可包含项目资料、逐图方案、参考图、任务信息和生成结果。
- 项目数据、任务记录和图片主要保存在当前浏览器本地。

## 2. 使用前准备

推荐环境：

- Windows 10 或 Windows 11
- Microsoft Edge 或 Google Chrome
- Node.js 20 LTS 或更新版本
- npm
- 可用的 AI 策划接口和图片生成接口

检查 Node.js 和 npm：

```powershell
node --version
npm --version
```

如果命令无法识别，请先安装 Node.js，再重新打开终端。

## 3. 启动和关闭程序

### Windows 推荐方式

下载或克隆项目后，打开项目根目录，双击：

```text
start-amazon-image-studio.bat
```

脚本会检查依赖、启动本地服务并打开浏览器。

停止程序时双击：

```text
stop-amazon-image-studio.bat
```

### 手动启动

在项目目录打开 PowerShell：

```powershell
npm ci
npm run dev
```

然后打开终端显示的本地访问地址。

手动停止时，在运行服务的终端按 `Ctrl + C`。

## 4. 第一次配置 API

程序本身不内置 API Key。AI 策划和图片生成会使用你自己的接口并产生相应费用。

### 4.1 打开设置

点击页面右上角的“设置”，进入 API 配置页面。

一份配置通常包括：

- 配置名称
- 服务商类型
- API 地址
- API Key
- 接口模式
- 模型 ID
- 超时时间及其他兼容参数

### 4.2 建议建立两类配置

#### AI 策划配置

用于：

- 整理商品事实；
- 生成标题、五点和通用文案；
- Amazon Listing / A+ 策划。

常用接口模式：

- Chat Completions
- Responses API

策划模型只输出文本，不一定能生成图片。

#### 图片生成配置

用于真正提交生图任务。

常用接口模式：

- Images API
- 支持图片输出的 OpenRouter Chat Completions 模型
- fal.ai
- 已导入的自定义图片服务商

普通文本聊天模型不能直接用于生图。如果点击生成时提示“当前配置不能生图”，请切换到支持图片输出的配置。

### 4.3 OpenAI 兼容接口示例

实际地址和模型必须以服务商文档为准：

```text
配置名称：我的生图接口
服务商类型：OpenAI 兼容接口
API 地址：https://你的服务商地址/v1
API Key：你的 Key
接口模式：Images API
模型 ID：服务商提供的图片模型 ID
```

### 4.4 DeepSeek 的注意事项

DeepSeek 官方 Chat Completions 更适合商品资料和文案策划，不是图片生成接口。当前程序会在使用官方 DeepSeek 策划时跳过参考图片，只发送文本资料，因此应把关键商品外观特征用文字写清楚。

### 4.5 API Key 安全

- 不要把 API Key 写进 README、代码或 Git 提交。
- 不要把包含 Key 的配置链接发给其他人。
- 配置通常保存在当前浏览器本地。
- 清理浏览器站点数据前，先在设置中导出备份。

## 5. 推荐的完整工作流程

建议按以下顺序操作：

1. 启动本地程序。
2. 在设置中确认 AI 策划配置和生图配置均可用。
3. 上传产品实拍、结构、包装或细节参考图。
4. 打开“商品资料助手”，只填写确定的信息，并明确哪些内容不能猜。
5. 检查事实卡，人工确认可靠推测，保留未知项。
6. 生成目标语言的商品文案。
7. 根据上架平台选择 Amazon、平台通用或自由创作模式。
8. 填写整套要求、目标平台和图片数量。
9. 生成或手动建立逐图方案。
10. 逐张检查用途、构图、文案、比例及额外要求。
11. 点击“确认整套方案”。
12. 选择某张图片，点击“填入生图栏”。
13. 在底部输入区再次检查最终提示词、参考图、尺寸、格式和数量。
14. 切换到真正支持生图的 API 配置并提交。
15. 在历史记录中检查结果，必要时复用、编辑或下载。
16. 多商品任务使用批量项目工作区。
17. 项目完成后导出 ZIP，按 SKU 保存到本地商品资料目录。

## 6. 商品资料助手

### 6.1 什么时候使用

当你只有以下部分资料时，先使用商品资料助手：

- 材质
- 尺寸
- 颜色
- 产品结构
- 使用场景
- 已确认卖点
- 包装清单的一部分
- 产品实拍图

不需要先人工编写完整标题和五点。

### 6.2 推荐输入格式

示例：

```text
产品是一款黑色可折叠旅行收纳袋。
已知材质：涤纶。
展开尺寸：45 × 32 × 16 cm。
已知用途：旅行衣物和日常物品收纳。
外观：黑色，带拉链和提手。

未知内容：容量、承重、防水等级、包装数量。
请不要猜测未知参数，不要编造认证、测试结果或对比数据。
```

输入原则：

- “已知”和“未知”分开写；
- 数字带单位；
- 不确定的信息直接写“不确定”；
- 明确要求 AI 不得推测认证、性能、容量、承重和包装数量；
- 参考图无法证明的内部材质或性能，不要写成事实。

### 6.3 整理商品事实

点击“整理商品事实”后，程序会产生：

- 已确认事实：来自用户描述或参考图明确支持的内容；
- 待确认推测：AI 判断可能成立，但仍需要用户确认；
- 缺失信息：目前没有资料；
- 矛盾信息：不同输入之间存在冲突；
- 暂定商品分类。

只有你勾选确认的推测才应进入后续商品事实。不要为了让文案看起来丰富而确认无法核实的信息。

### 6.4 生成商品文案

填写输出语言，例如：

```text
English (United States)
German
French
Spanish
Japanese
```

点击“生成商品文案”，当前版本会生成：

- Amazon 标题
- Amazon 五点
- 通用简短描述
- 通用详细描述
- 通用卖点

文案生成后仍需人工检查：

- 尺寸和单位是否正确；
- 单复数和包装数量是否正确；
- 是否出现未经证实的性能词；
- 是否出现绝对化、医疗、认证或对比声明；
- 是否符合目标站点语言习惯。

## 7. Amazon 合规模式

Amazon 模式适合制作 Listing 图片和 A+ Content。

### 7.1 Listing 工作流

1. 选择“Amazon 合规”。
2. 打开商品资料助手，整理事实并生成 Amazon 文案。
3. 将文案回填到 Amazon 策划区，或手动输入标题和五点。
4. 上传产品参考图。
5. 设置 Listing 图片数量和目标尺寸。
6. 使用 AI 策划生成 MAIN、PT01、PT02 等图片位方案。
7. 逐个选择图片位，检查提示词预览。
8. 添加整套要求和当前图片额外要求。
9. 将当前图片填入生图区域并提交。

### 7.2 A+ 工作流

1. 在 Amazon 策划中切换到 A+。
2. 选择普通 A+、标准 A+、高级 A+ 或手机 A+。
3. 检查右侧模块编排和推荐尺寸。
4. 按需要增加或删除同规格模块。
5. 生成 A+ 策划。
6. 逐个模块生成图片，并把较长正文留给 Seller Central 文本区域。

### 7.3 Amazon 主图限制

Amazon MAIN 主图会优先执行硬规则，例如：

- 纯白背景；
- 商品完整、比例真实；
- 不添加水印、促销文案、边框或平台徽章；
- 不使用 Amazon、Prime、Best Seller、评分、价格等风险元素；
- 不得改变商品结构、颜色、数量、包装内容和已确认事实。

如果确实需要水印、Logo、海报文字或更自由的构图，请切换到“自由创作”，不要尝试在 Amazon MAIN 主图中强行覆盖平台规则。

## 8. 平台通用模式

平台通用模式适合非 Amazon 渠道。

### 使用步骤

1. 选择“平台通用”。
2. 填写目标平台，例如 Walmart、eBay、Temu 或独立站。
3. 填写整套要求。
4. 在“平台补充说明”中填写你已经确认的平台尺寸、文字和上架要求。
5. 设置图片数量。
6. 点击“按描述生成方案”，或手动新增图片卡片。
7. 逐张编辑图片用途、画面目标、构图、图片文案、比例和额外要求。
8. 确认整套方案。
9. 选择一张图片并填入生图栏。

平台通用模式不会自动继承 Amazon 主图白底、固定图片位和禁止水印等专属规则，但仍会执行商品真实性和自有素材要求。

## 9. 自由创作模式

自由创作适合：

- 品牌海报
- 社交媒体图片
- 独立站视觉图
- 带自有 Logo 的商品图
- 带自有水印的图片
- 自定义比例和构图的场景图

### 使用步骤

1. 选择“自由创作”。
2. 根据需要开启“图片文字”“自有 Logo”“自有水印”等权限。
3. 在“整套创作要求”中直接说明数量和画面要求。
4. 生成或手动编辑逐图方案。
5. 对每张图写清楚文字位置、Logo 位置、水印透明度、人物、背景和道具。
6. 确认方案后填入生图栏。

示例：

```text
生成 4 张图片，整套保持暖色户外旅行风格。
第 1 张为干净的商品主视觉；
第 2、3 张为不同旅行场景；
第 4 张为 9:16 竖版海报。
右下角加入我方自有品牌 Logo，保持小尺寸，不遮挡商品。
所有商品外观、拉链数量、提手结构和颜色必须与参考图一致。
```

自由模式提高的是创作自由度，不代表可以绕过服务商安全政策、版权规则或他人商标权。

## 10. 逐图方案编辑器

### 10.1 图片卡片字段

每张图片可以设置：

- 图片用途
- 画面目标
- 构图
- 图片文案
- 比例：1:1、4:5、3:4、16:9、9:16
- 单张额外要求
- 方案内部的分辨率、格式和风格信息

### 10.2 卡片操作

- 新增图片：最多 12 张；
- 复制：复制当前卡片后再修改；
- 上移 / 下移：调整套图顺序；
- 删除：至少保留 1 张；
- 点击顶部卡片：切换当前编辑图片。

### 10.3 方案确认

点击“确认整套方案”后，当前方案进入已确认状态。

以下操作会使确认失效：

- 修改用途、目标、构图、文案或要求；
- 修改图片数量；
- 新增、复制、删除或移动卡片；
- 重新生成整套方案。

确认失效后，“填入生图栏”会禁用，需要再次检查并确认。

### 10.4 提示词优先级

程序按以下顺序组织提示词：

1. 安全要求和当前模式硬规则；
2. 用户整套要求；
3. 当前图片额外要求；
4. 已确认商品事实；
5. 图片目标、构图、证据和文案；
6. 当前图片风格或整套风格；
7. 套图一致性；
8. 比例、分辨率和格式；
9. 排除项和负面要求。

用户额外要求的优先级高于默认风格，但不能覆盖商品真实性、素材授权和当前平台硬规则。

## 11. 参考套图重构

### 使用步骤

1. 在底部参考图区域上传一套图片。
2. 进入平台通用或自由创作模式。
3. 选择重构强度：轻度重构、标准重构或深度原创。
4. 点击“按 N 张参考图重构”。
5. 程序会按参考图数量建立一一对应的逐图方案。
6. 逐张修改用途、构图、文案、背景、人物和道具要求。
7. 确认整套方案后逐张生成。

### 三种重构方向

- 轻度重构：保留更多营销目的和信息层级，视觉变化相对温和；
- 标准重构：保留营销功能，但重新设计背景、人物、道具和构图；
- 深度原创：尽量降低与参考套图的视觉相似度，允许更大幅度重新策划。

### 推荐提示方式

```text
保留每张参考图的营销目的，但不要复制原图文字、Logo、固定版式、人物姿势和背景布置。
商品主体必须以我上传的产品实拍为准。
所有图片重新生成文案；如无法确认卖点，则不在图片中展示。
人物、道具和场景允许重新设计，但不能遮挡商品关键结构。
```

### 人工风险检查

生成后重点检查：

- 是否残留第三方 Logo 或平台徽章；
- 是否复用了原图句子；
- 构图、人物姿势和道具位置是否过于相似；
- 包装和产品结构是否被改错；
- 自有 Logo 是否正确；
- 图片中的尺寸、数量和性能信息是否真实。

## 12. 批量项目工作区

批量工作区提供 5 个独立项目槽位。

### 12.1 填写项目

每个项目单独填写：

- 项目名称
- SKU / 标记
- 商品材质、尺寸和卖点
- 整套生图要求
- 图片数量（1–12）
- 参考图（最多 6 张）

不同项目的数据和参考图互不覆盖。

### 12.2 执行队列

1. 勾选需要生成的项目。
2. 可使用“全选”或只选择部分项目。
3. 点击“执行队列”。
4. 程序为每个项目建立逐图方案并提交生图任务。

默认并发：

- 最多同时处理 2 个项目；
- 每个项目最多同时提交并等待 2 张图片；
- 因此批量工作区最多同时运行 4 个图片任务。

### 12.3 状态说明

| 状态 | 含义 |
| --- | --- |
| 待配置 | 尚未执行 |
| 排队中 | 已选择，等待进入执行器 |
| 生成中 | 已经创建并运行图片任务 |
| 已提交，等待任务记录 | 页面正在恢复历史任务信息 |
| 未完成，可继续 | 刷新或中断前还有图片未提交完成 |
| 已完成 | 本项目计划图片均已成功完成 |
| 生成失败 | 至少一张已提交图片失败 |

### 12.4 失败和刷新恢复

- 单个图片失败不会停止其他项目。
- 刷新页面后，已创建的任务编号会继续和历史记录对应。
- 如果部分图片尚未提交，项目显示“未完成，可继续”。
- 点击“继续未完成图片”时，只生成缺失或失败的图片。
- 已成功的图片不会重新提交，避免重复消耗额度。

### 12.5 载入单个项目

点击“载入 项目名称”，可以把该项目的说明和参考图载入主生图工作区，便于人工检查和单独调整提示词。

## 13. 生图输入区与参数

在真正提交前，检查以下内容：

- 当前 API 配置是否支持生图；
- 提示词是否对应当前图片；
- 参考图是否属于当前商品；
- 图片尺寸和比例是否合适；
- 输出格式是否正确；
- 生成数量是否正确；
- 是否误带其他项目的 Logo、包装或参考图。

### 参考图建议

- 至少上传一张清晰正面或三分之四角度产品图；
- 补充背面、侧面、结构、配件和包装图；
- 同一组参考图不要混入不同 SKU；
- 大图会在请求前做兼容处理，但仍应避免无意义的超大文件；
- 参考图用于保持产品一致性，不代表模型一定能准确还原所有小字。

### 数量和并发

普通生图区域的数量参数由当前服务商能力决定。批量项目为了便于管理，会按逐图任务执行，不依赖一次请求返回整套图片。

## 14. 历史记录和结果复用

生成任务会进入历史记录。

常用操作：

- 查看成功、运行中和失败任务；
- 查看任务实际提示词和参数；
- 下载单张或批量图片；
- 收藏重要结果；
- 复用已有任务配置；
- 将输出图作为新参考图继续编辑；
- 对失败任务进行重试；
- 按工作流、来源和图片比例筛选。

复用历史任务时，应检查当前 API 配置是否仍然存在。如果原配置被删除，程序会提示是否改用当前配置。

## 15. ZIP 项目归档

在批量工作区点击“导出 ZIP”。

ZIP 文件名使用日期和 SKU / 项目名称，例如：

```text
2026-07-13-SKU-001.zip
```

可能包含：

```text
project.json
README.txt
references/
  reference-01.png
  reference-02.jpg
generated/
  task-01-image-01.png
  task-02-image-01.webp
```

`project.json` 保存项目字段、逐图方案、任务状态和对应关系；`references` 保存参考图；`generated` 保存能够从本地任务记录读取到的生成结果。

建议本地目录结构：

```text
电商项目/
  2026-07/
    SKU-001-产品名称/
      原始资料/
      参考图/
      AI生成图/
      精修图/
      上架终稿/
      2026-07-13-SKU-001.zip
```

## 16. 本地数据和备份

程序主要使用浏览器本地存储和 IndexedDB 保存：

- API 配置
- 商品事实卡和文案
- 模式草稿
- 逐图方案
- 批量项目资料
- 任务历史
- 参考图和生成图

需要注意：

- 换浏览器、换电脑或使用无痕模式，不会自动同步数据；
- 清除站点数据可能删除本地记录和图片；
- 浏览器存储空间有限，大量高分辨率参考图可能导致批量项目草稿无法完整持久化；
- 重要项目完成后应立即下载图片并导出 ZIP；
- 定期使用设置中的数据导出功能备份。

## 17. 常见问题

### 17.1 双击启动没有打开页面

检查：

1. Node.js 是否安装；
2. 5173 端口是否被其他程序占用；
3. 启动窗口中是否显示依赖安装失败；
4. 手动打开终端显示的本地访问地址。

### 17.2 提示当前配置不能生图

当前配置可能是 Chat Completions 或 Responses 文本策划配置。进入设置，切换到 Images API、fal.ai、支持图片输出的 OpenRouter 模型或有效自定义图片服务商。

### 17.3 商品资料助手不能读取参考图

如果使用 DeepSeek 官方策划接口，程序会只发送文字。请把商品外观、颜色、结构、材质和尺寸写入描述，或改用支持视觉输入的策划接口。

### 17.4 AI 忽略我的水印或文字要求

先确认当前模式：

- Amazon MAIN 主图会拒绝水印和促销文字；
- 平台通用或自由创作允许自有 Logo / 水印，但仍受图片服务商能力影响；
- 把要求写到“当前图片额外要求”，并写清位置、大小、透明度和内容；
- 小字和复杂 Logo 可能需要后期设计软件补充。

### 17.5 生成图片格式过于固定

使用平台通用或自由创作模式，不要使用 Amazon 固定图片位；修改每张图片的用途、比例、构图和额外要求，并在生成前确认整套方案。

### 17.6 批量项目刷新后没有继续自动生成

刷新会恢复已经创建的任务状态，但为了避免重复扣费，不会盲目重新提交未确认的任务。项目显示“未完成，可继续”时，人工点击“继续未完成图片”。

### 17.7 ZIP 中没有生成图片

可能原因：

- 项目尚未完成；
- 任务失败或输出图已被删除；
- 浏览器本地图片记录被清理；
- 项目是在任务关联功能加入前创建的旧数据。

先检查历史记录中是否还能看到生成结果，再重新导出。

### 17.8 浏览器换了以后数据不见了

本地数据不会自动跨浏览器同步。请在原浏览器导出备份，再在新浏览器导入。

## 18. 合规与版权提醒

本工具用于辅助策划、生成和整理，不提供法律保证。

使用者需要自行确认：

- 上传的产品图片、Logo、水印、字体和人物素材拥有使用权；
- 不复制竞争对手文案、商标、包装、平台徽章和固定版式；
- 生成图中的商品结构、数量、尺寸、颜色和包装内容真实；
- 不使用未经证明的认证、医疗、性能和对比声明；
- 上架前人工检查目标平台最新规则。

“无侵权”“绝对原创”无法仅靠 AI 自动保证。正确做法是减少相似风险、保存创作过程、使用自有素材，并在重要商业项目中进行人工或专业审核。

## 19. 当前版本边界

为避免误解，请注意当前实现范围：

1. 商品事实分析和商品文案会调用已配置的 AI 策划接口。
2. 平台通用 / 自由模式的“按描述生成方案”目前根据图片数量和本地用途模板建立结构化方案，不是完整的远程 LLM 自动策划。
3. “参考图重构”目前按参考图数量建立一一对应方案，并把重构要求写入提示词；它不会先运行独立计算机视觉模型逐张识别营销结构。
4. 相似风险检查目前以人工检查清单为主，不是法律鉴定或自动侵权检测。
5. 批量工作区内部最多同时运行 4 个图片任务，但其他手动生图或 Agent 任务不计入该批量上限。
6. 当前项目归档以 ZIP 为主，尚未自动选择本地根目录并创建完整文件夹树。
7. 实际图片质量、文字准确度、Logo 还原和参考图一致性取决于所选模型及服务商。
